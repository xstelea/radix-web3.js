# @radix-effects/sbor

`@radix-effects/sbor` provides Effect Schema transforms for Gateway
`ProgrammaticScryptoSborValue` objects. It is useful when Gateway returns
programmatic SBOR for events, component state, key-value store values, or
non-fungible data and you want decoded values with Radix meaning preserved.

## Install

```bash
npm install @radix-effects/sbor
```

## Basic Usage

```ts
import { Effect, Schema } from 'effect';
import { bool, decimal, resourceAddress, struct } from '@radix-effects/sbor';

const SwapEvent = struct({
  input_address: resourceAddress,
  input_amount: decimal,
  output_address: resourceAddress,
  output_amount: decimal,
  is_success: bool,
});

const program = Effect.gen(function* () {
  const parsed = yield* Schema.decodeUnknown(SwapEvent)(programmaticSborValue);
  console.log(parsed.input_amount.toString());
});
```

The exported schemas are ordinary Effect schemas. Decode Gateway SBOR with
`Schema.decodeUnknown(schema)` and encode decoded values back with
`Schema.encode(schema)`.

## Native Schemas

| Export | Decodes |
| --- | --- |
| `value` | Raw Gateway SBOR value as a fallback. |
| `string` | SBOR string values. |
| `bool` | Boolean values. |
| `bytes` | SBOR bytes as a hex string. |
| `u8`, `u16`, `u32`, `u64`, `u128` | Explicit unsigned integers as `BigNumber`. |
| `i8`, `i16`, `i32`, `i64`, `i128` | Explicit signed integers as `BigNumber`. |
| `decimal` | `Decimal` as `BigNumber`. |
| `preciseDecimal` | `PreciseDecimal` as `BigNumber`. |
| `number` / `numeric` | Any numeric SBOR value as `{ type, value: BigNumber }`. |
| `instant` | Radix `Instant` values as `Date`. |
| `resourceAddress`, `componentAddress`, `accountAddress`, `packageAddress` | Referenced addresses as shared branded types. |
| `internalAddress`, `vaultAddress`, `keyValueStoreAddress` | Owned addresses as shared branded types. |
| `nonFungibleLocalId` | Non-fungible local id as the shared branded type. |
| `struct(fields)` | Field-name based tuple/struct transform. |
| `tuple([...schemas])` | Ordered tuple transform. |
| `array(schema)` | Homogeneous array transform. |
| `option(schema)` | Rust `Option` enum transform. |
| `map({ key, value })` | SBOR map transform into a readonly map. |
| `enumeration(variants)` | Rust-style enum transform. |

## Type Inference

```ts
import { Schema } from 'effect';
import { decimal, string, struct, u64 } from '@radix-effects/sbor';

const schema = struct({
  pair: string,
  amount: decimal,
  updated_at: u64,
});

type Parsed = Schema.Schema.Type<typeof schema>;
```

The inferred type follows the schema. For the example above, `Parsed` contains
`pair: string`, `amount: BigNumber`, and `updated_at: BigNumber`.

## Parsing Nested Values

```ts
const schema = struct({
  updates: array(
    tuple([
      string,
      struct({
        oi_long: decimal,
        oi_short: decimal,
        last_update: u64,
      }),
    ]),
  ),
});
```

Use `tuple` for ordered SBOR tuple fields and `struct` when the Gateway
value carries field names.

## Enums And Options

```ts
const balanceChange = enumeration([
  {
    variant: 'NonFungible',
    schema: struct({
      ids: array(nonFungibleLocalId),
      resource_address: resourceAddress,
    }),
  },
  {
    variant: 'Fungible',
    schema: tuple([decimal]),
  },
]);

const maybeName = option(string);
```

Enum parses return `{ variant, value }`. Option parses return
`{ variant: 'Some', value }` or `{ variant: 'None' }`.
